# LesaPay (レサペイ)

家庭内報酬管理アプリ。子どもが日々の目標を達成 → スマホから完了報告 → 親が承認 → ポイントが貯まる → 親が換金（お小遣い化）する一連の流れを、Firebase Hosting + Google Apps Script + Google スプレッドシートで運用します。

**1端末1人運用**: 1つのスプレッドシート + 1つのGASを家族で共用し、子ごとにシートを分けます。各端末は自分のシート名だけを覚えていて、その分のデータしか触れません。GitHubに置くコードには個人情報は一切含まれません。

## 構成

```
lesa-pay/
├── public/             # Firebase Hosting にデプロイするフロント一式
│   ├── index.html
│   ├── css/style.css
│   └── js/
│       ├── config.js   # localStorage キー定義のみ (個人情報なし)
│       └── app.js      # アプリ本体
├── gas/Code.gs         # Google Apps Script バックエンド
├── firebase.json       # Firebase Hosting 設定 (public/ をデプロイ)
└── .firebaserc
```

## セットアップ

### 1. スプレッドシート作成（家族で1つ）

1. 新しい Google スプレッドシートを作成（例: `LesaPay`）
2. シートID（URL の `/d/` と `/edit` の間）を控える

子ごとに「課題_◯◯」「履歴_◯◯」の2シートを並べます（GASの `setupSheets("名前")` で自動作成）:

| シート名 | A | B | C | D | E | F | G |
| - | - | - | - | - | - | - | - |
| 課題_ライト | ID | 科目 | 分類 | 項目 | 報酬ポイント | ステータス | 有効期限 |
| 履歴_ライト | 日時 | 内容 | ポイント | | | | |
| 課題_ティアラ | (同上) | | | | | | |
| 履歴_ティアラ | (同上) | | | | | | |

子が増えたら同じ要領でシートを2枚追加するだけ。

### 2. Google Apps Script（家族で1つ）

1. [script.google.com](https://script.google.com/) で新規プロジェクト作成
2. `gas/Code.gs` の内容を貼り付けて保存
3. **プロジェクトの設定 → スクリプトプロパティ** に登録:

   | プロパティ | 値 |
   | - | - |
   | `SHEET_ID` | スプレッドシートID |
   | `PARENT_PASSWORD` | 保護者パスワード |

4. 子ごとにシートを作成する。以下のどちらでもOK:
   - **手動**: スプレッドシートで `課題_ライト` `履歴_ライト` という名前のシートを直接2枚作り、ヘッダー行を入れる
   - **GASで自動**: GASエディタで一時関数を書いて実行
     ```js
     function tmp() { setupSheets('ライト'); }
     ```
     `tmp` を選んで ▶️ 実行（初回のみ承認ダイアログ）。終わったら `tmp` は消してOK

5. **デプロイ → 新しいデプロイ → ウェブアプリ**
   - 実行ユーザー: **自分**
   - アクセス権: **全員**
6. 発行されたURLを控える

### 3. アプリ起動

ブラウザで `index.html` を開く（ローカル: `python3 -m http.server 8080`、または Firebase Hosting にデプロイ）。

初回起動時にセットアップ画面が出るので入力:

- **GAS Web App URL**: ステップ2で控えたURL
- **シート名**: その端末で使う子のシート名サフィックス (例: 姉の端末なら `ライト`)
- **愛称**(任意): 画面に表示する名前。空ならシート名と同じ

設定は端末のlocalStorageに保存されます。

設定を変更したいときは、ヘッダーの **⚙️ ボタン** から再設定できます。

## 運用フロー

1. **課題の追加** (親) — スプレッドシートの「課題_◯◯」シートに行を追加。A列(ID)とF列(ステータス)は **空欄でOK**
2. **完了報告** (子) — アプリの「完了報告」ボタン → ステータスが `申請中` に
3. **承認** (親) — アプリ右上🔑 → パスワード入力 → 申請中の課題に「✓承認」「✗却下」ボタン。承認すると履歴に自動記録
4. **換金** (親) — 保護者モードで「換金する」 → 履歴に `-XXX` で記録

> 親が両方の子を見たい場合: PCブラウザを2つ開く / シークレットウィンドウ / 別プロファイル などで別々のシート名を設定して切り替えられます。あるいは ⚙️ から都度シート名を変更してもOK。

## セキュリティ

- 保護者パスワードはGAS側のみで検証 (フロントは入力値をメモリに持つだけ)
- スプレッドシート・GASプロジェクトの共有設定は **自分のみ**
- リポジトリには個人情報・URLを含めない
